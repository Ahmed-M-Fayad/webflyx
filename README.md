# webflyx

A hands-on Git practice project completed as part of the [Boot.dev Learn Git](https://www.boot.dev/courses/learn-git) course.

## About

webflyx is a fictional movie streaming catalog used as a playground for practicing core Git workflows. The repository content is intentionally simple — the focus is entirely on the Git operations performed on it, not the data itself.

## What Was Practiced

- Initializing and configuring a local repository
- Staging, committing, and writing structured commit messages
- Branching, merging, and resolving merge conflicts
- Rebasing and understanding rebase vs merge tradeoffs
- Working with a GitHub remote: push, pull, fetch
- Using `.gitignore` to exclude file types from tracking

## Repository Structure

```
webflyx/
├── titles.md          # Movie titles in the WebFlyx collection
├── classics.csv       # Classic movies with director and year
├── contents.md        # Index of repository files
├── advert.md          # Promotional content
├── quotes/
│   ├── dune.md        # Memorable quotes from Dune
│   └── starwars.md    # Memorable quotes from Star Wars
└── .gitignore         # Excludes .html files
```
